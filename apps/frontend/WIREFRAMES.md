## Accessibility Notes

- All interactive elements keyboard accessible (Tab, Enter, Esc)
- ARIA labels for screen readers
- Sufficient color contrast (WCAG AA minimum)
- Focus indicators visible
- Form validation announces errors
- Modal focus trapping
- Skip to content link

---

## Notes for Implementation

1. **Use Tailwind CSS** for rapid UI development
2. **Component-first approach**: Build reusable components for common patterns
3. **Responsive by default**: Mobile-first design
4. **Loading states**: Show skeletons or spinners during data fetch
5. **Empty states**: Friendly messages when lists are empty
6. **Error states**: Clear error messages with recovery actions
7. **Optimistic updates**: Update UI immediately, revert on error
8. **Animations**: Subtle transitions for better UX (Tailwind transitions)
9. **Icons**: Use a consistent icon library (Heroicons recommended)
10. **Forms**: Use controlled components with validation

## Figma / Design Tool Integration

When creating actual mockups in Figma or similar:
- Create component library matching Tailwind's design tokens
- Use Auto Layout for responsive components
- Maintain design system documentation
- Create interactive prototypes for user testing
- Export assets at 1x, 2x, 3x for different screen densities
